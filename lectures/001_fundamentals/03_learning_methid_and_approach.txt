คอวพิวเตอร์เป็นเครื่องจักร์ที่โง่มาก แต่สามารถคำนวนได้รวดเร็ว
เพราะฉนั้นเราจะต้องเป็นคนสั่งคอมพิวเตอร์ ว่าจะต้องทำงานอะไร และทำอย่างไร

ดังนั้นเราจึงได้ออกแบบชุดคำสั่ง (instruction set) สำหรับให้คอมพิวเตอร์ทำงาน

CPU (Central Processing Unit)
    เป็นตัวประมวลผลของคอมพิวเตอร์ ทุกคำสั่งที่เราสั่งคอมพิวเตอร์ให้ทำงาน จะถูกส่งมาประมวลผลที่นี่

RAM (Random Access Memory)
    ใช้จัดเก็บข้อมูลของชุดคำสั่ง ที่กำลังทำงานอยู่ในปัจจุบัน
    โดยข้อมูลจะหายไป เมื่อไม่มีการจ่ายไฟฟ้า

Hard drive
    ใช้จัดเก็บข้อมูล โดยอาจเป็น ชุดคำสั่ง ทรัพยากรณ์ และจะข้อมูลจะไม่หายไป แม้จะไม่มีการจ่ายไฟ

Operating System
    ระบบปฏิบัติการ ช่วยจัดการการทำงานระหว่างอุปกรณ์ต่าง ๆ ในคอมพิวเตอร์ให้
    ทำให้ผู้ใช้สามารถใช้งานคอมพิวเตอร์ได้ง่ายขึ้น
    เข่น 
        - Windows
        - Linux
        - MacOS

Fetch / Execute cycle (life of a CPU)
    cpu จะทำการดึงชุดคำสั่งที่จะต้องทำงานอยู่ตลอด โดยจะดึงจาก memory และนำชุดคำสั่งที่ได้มาประมวลผล เมื่อเสร็จแล้วจะวนไปดึงคำสั่งใหม่
    โดยความเร็วในการวนซ้ำ จะขึ้นอยู่กับจำนวน gigahertz (GHz) เช่น CPU INTEL CORE I-5 5400F ความเร็ว 3.4 GHz

Life Cycle 
    ประกอบด้วย 3 ตัว
        1. Control Unit
        2. ALU
        3. Register
    โดยมีขั้นตอนการทำงานแบบนี้
        1. ALU              ---> [fetch]        ---> Register
        2. Register         ---> [Decode]       ---> Control Unit
        3. Control Unit     ---> [Execute]      ---> ALU


High level programming language
    เป็นภาษาโปรแกรมที่ออกแบบมาให้มนุษย์สามารถเข้าใจได้ง่าย
    โดยแต่ก่อน การจะสั่งงาน cpu จะต้องเขียนเป็น assembly language ซึ่งยากมากต่อการเข้าใจ

    โดยภาษา c เป็น high level programming โดยการเขียนภาษา c เราไม่จำเป็นต้องจำคำสั่งต่าง ๆ ที่เอาไว้สั่ง cpu โดยตรง

Compiler
    ทำหน้าที่แปรง High level programming เป็น Low level programming เพื่อให้ cpu สามารถเข้าใจชุดคำสั่งได้

ขั้นตอนการพัฒนาโปรแกรมจะมีดังนี้
    1. define the program objective
    2. design the program 
    3. write the code
    4. compile 
    5. run the program 
    6. test and debug the program 
    7. maintain and modify the program
